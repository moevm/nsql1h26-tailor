## Видение проекта:

### Пользовательские роли

Для роли заказчика:
- Создание нового заказа, указав необходимые параметры, такие как тип услуги, описание заказа, предпочтительные сроки выполнения и контактную информацию.
- Отслеживание статуса заказа, видеть предполагаемое время выполнения и получать уведомления о любых изменениях статуса заказа.
- Возможность просматривать историю своих заказов и оставлять отзывы о выполненных заказах.

Для роли работника:
- Получение задач, назначенных ему, и возможность фильтровать их по статусу (например, "новые", "в процессе", "готово").
- Работник должен иметь возможность просматривать детали заказа, включая требования заказчика, и принимать или отклонять заказ.
- После принятия заказа, работник должен иметь возможность обновлять его статус (например, "в процессе", "готово") и добавлять комментарии или фотографии для информирования заказчика о ходе выполнения заказа.

Для роли руководства:
- Руководство должно иметь доступ к панели управления, где можно управлять сотрудниками.
- Смотреть финансовую аналитику.
- Составлять отчёты.

### Коллекции
users:
- id
- name
- email
- password
- role (customer, worker, management)
- createdAt

orders:
- id
- customerId (ссылка на пользователя, который создал заказ)
- workerId (ссылка на пользователя, который выполняет заказ)
- status (новый, в процессе, готово)
- items (список услуг или товаров, связанных с заказом)
- createdAt
- updatedAt
- details (текстовое описание заказа, требования заказчика)

tasks:
- id
- orderId (ссылка на заказ, к которому относится задача)
- description (текстовое описание задачи)
- status (новый, в процессе, готово)
- createdAt
- updatedAt

materials:
- id
- orderId (ссылка на заказ, для которого используются материалы)
- name (название материала)
- quantity (количество материала)
- cost (стоимость материала)
- createdAt
- updatedAt

payments:
- id
- orderId (ссылка на заказ, для которого произведён платёж)
- amount (сумма платежа)
- paymentDate (дата платежа)
- createdAt
- updatedAt
